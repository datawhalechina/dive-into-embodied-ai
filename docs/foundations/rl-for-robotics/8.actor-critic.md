# 8. Actor-Critic 算法

纯策略梯度算法（如 REINFORCE）虽然解决了连续动作空间和随机策略的问题，但高方差和低采样效率限制了其在复杂任务中的表现。$\text{Actor-Critic}$ 算法将策略函数和值函数同时参数化，兼顾两者的优点。

> **具身智能视角**：Actor-Critic 是 PPO、SAC、TD3 等主流算法的共同基础框架。理解 Actor-Critic 的原理（特别是 GAE）对于调试和改进具身智能中的 RL 训练至关重要。

## 8.1 纯策略梯度的缺点

策略梯度算法的目标函数：

$$
\begin{equation}
\nabla_\theta J(\theta) \propto \mathbb{E}_{\pi_{\theta}}\left[\Psi_t \nabla_\theta \log \pi_\theta(a_t \mid s_t)\right]
\end{equation}
$$

其中 $\Psi_t$ 使用蒙特卡洛估计（$\Psi_t = G_t$）时，方差高、采样效率低，在奖励稀疏的环境中问题尤其严重。

## 8.2 Actor-Critic 原理

Actor-Critic 将值估计交给独立的 Critic 网络，而 Actor 专注于策略优化：

<Figure src={require('./figs/actor_critic_architecture.webp').default} caption="Actor-Critic 算法架构" />

Actor-Critic 更多是一种**框架**，不同算法的实现可能不同。对于 Critic 部分，可以使用状态价值函数 $V^\pi(s_t)$ 或动作价值函数 $Q^\pi(s_t, a_t)$。

**Value Actor-Critic**（使用状态价值）：

$$
\begin{equation}
\nabla_\theta J(\theta) \propto \mathbb{E}_{\pi_{\theta}}\left[V_\omega(s_t) \nabla_\theta \log \pi_\theta(a_t \mid s_t)\right]
\end{equation}
$$

Critic 通过时序差分（TD）方法更新参数：

$$
\begin{equation}
y_i = r_i + \gamma V_\omega(s_{i+1}) - V_\omega(s_i), \quad
\omega \leftarrow \omega + \beta y_i \nabla_\omega V_\omega(s_i)
\end{equation}
$$

## 8.3 A2C 算法

为了进一步降低方差，引入**优势函数（Advantage Function）**：

$$
\begin{equation}
A^\pi(s_t, a_t) = Q^\pi(s_t, a_t) - V^\pi(s_t)
\end{equation}
$$

优势函数衡量了在给定状态下选择特定动作相对于平均水平的优势。通过减去基线 $V^\pi(s_t)$，减少了梯度估计的方差。

A2C 的目标函数：

$$
\begin{equation}
\nabla_\theta J(\theta) \propto \mathbb{E}_{\pi_{\theta}}\left[A^\pi(s_t, a_t) \nabla_\theta \log \pi_\theta(a_t \mid s_t)\right]
\end{equation}
$$

## 8.4 A3C 算法

A3C（Asynchronous Advantage Actor-Critic）通过多个并行智能体同时与环境交互，每个智能体将参数异步更新到全局网络：

<Figure src={require('./figs/a3c_architecture.webp').default} caption="A3C 算法架构" />

多进程训练的思路也广泛应用于其他算法（如 PPO 在 Isaac Gym 中使用数千个并行环境），能显著提高训练效率和探索能力。

## 8.5 广义优势估计（GAE）

优势函数的估计方式直接影响训练效果。常见的估计方式有：

**单步 TD 估计**（方差小、偏差大）：

$$
\begin{equation}
\hat{A}_t^{\mathrm{TD}} = r_t + \gamma V^\pi(s_{t+1}) - V^\pi(s_t)
\end{equation}
$$

**蒙特卡洛估计**（无偏、方差大）：

$$
\begin{equation}
\hat{A}_t^{\mathrm{MC}} = \sum_{l=0}^{\infty} \gamma^l r_{t+l} - V^\pi(s_t)
\end{equation}
$$

**广义优势估计（GAE）** 通过参数 $\lambda$ 在两者之间权衡：

$$
\begin{equation}
A^{\mathrm{GAE}(\gamma, \lambda)}_t = \sum_{l=0}^{\infty}(\gamma \lambda)^l \delta_{t+l}
\end{equation}
$$

其中 $\delta_{t+l} = r_{t+l} + \gamma V^\pi(s_{t+l+1}) - V^\pi(s_{t+l})$ 是 TD 误差。

GAE 有一个高效的递推形式，非常适合实现：

$$
\begin{equation}
A^{\mathrm{GAE}(\gamma, \lambda)}_t = \delta_t + \gamma \lambda A^{\mathrm{GAE}(\gamma, \lambda)}_{t+1}
\end{equation}
$$

- 当 $\lambda = 0$ 时，退化为单步 TD 误差
- 当 $\lambda = 1$ 时，退化为蒙特卡洛估计
- 常用 $\lambda = 0.95$，在偏差和方差之间取得平衡

### 8.5.1 GAE 的 PyTorch 实现

```python
import torch

def compute_gae(rewards, values, dones, gamma=0.99, lam=0.95):
    """
    计算 Generalized Advantage Estimation (GAE)
    rewards: shape = [T]，每一步的即时奖励
    values:  shape = [T+1]，值函数 V(s_t)
    dones:   shape = [T]，是否为终止状态
    """
    T = len(rewards)
    advantages = torch.zeros(T, dtype=torch.float32)
    last_adv = 0.0

    for t in reversed(range(T)):
        if dones[t]:
            next_non_terminal = 0.0
            next_value = 0.0
        else:
            next_non_terminal = 1.0
            next_value = values[t + 1]

        delta = rewards[t] + gamma * next_value * next_non_terminal - values[t]
        advantages[t] = delta + gamma * lam * next_non_terminal * last_adv
        last_adv = advantages[t]

    returns = advantages + values[:-1]
    return advantages, returns
```

## 8.6 小结

Actor-Critic 框架是现代 RL 算法的基础：

1. **Actor** 负责策略优化，**Critic** 负责价值评估
2. **优势函数**通过减去基线降低方差
3. **GAE** 提供了偏差-方差权衡的灵活方式
4. PPO = Actor-Critic + 重要性采样 + Clip 约束
5. SAC = Actor-Critic + 最大熵 + 自动温度调节
